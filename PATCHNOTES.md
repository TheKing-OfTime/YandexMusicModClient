## Исправлено
- Исправлена проблема из за которой при включённом кроссфейде приложение переставало передавать в OS метаданные текущего трека. (в том числе переставали работать медиа-клавиши).
  - Это происходило из за стрёмного API Хрома который:
    - Молча игнорирует метаданные если нет активного плеера или если их несколько.
    - Кеширует последнее значение метаданных и не отправляет их даже если по факту метаданные были проигнорированы при прошлой отправке.
  - Реализация в ЯМ не учитывала эти особенности и из за этого возникала проблема.
  - Мод теперь исправляет эту проблему. Те кто пользовался временным фиксом через отключение эксперимента `WebNextCrossMediaPlayer` могут убрать переопределение этого флага.
